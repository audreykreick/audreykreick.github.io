The term semantic refers to the meaning of the text. Writing semantic HTML means using HTML elements to structure your content based on each element's meaning, not its appearance. A semantic element clearly defines its meaning to both the developer and the browser. The purpose of using semantic tags is to organize the script based on the meaning of the HTML element, instead of how it looks. Using these tags can also help someone without coding knowledge understand why a piece of content was placed in a certain spot. Benefits of Using Semantic Tags

  • Search Engine Optimisation (SEO): HTML semantic elements are included in Google's algorithms. Search engines use HTML to decipher the content and goal of a webpage.
  • Accessibility: It is easier for everyone, including people with disabilities, to navigate a website if a foundation of semantic HTML is being used.
  • Browser Compatibility: HTML semantic tags make sure that your code can be integrated with future technologies.
